How It Works 運作原理
Three layers, one command 三個層次，一行指令
🔌
Collect 擷取
opencli + a Chrome extension borrow your browser's real session — no stored credentials. Search + detail + dropdown-click primitives run locally and return structured JSON. opencli + Chrome 擴充程式借用你本機 Chrome 的 session，不存任何帳密。search / detail / dropdown-click 都是本機 primitive，回傳結構化 JSON。
🧠
Recover 修補
When a platform's DOM doesn't match the selectors (redesign, new product type, A/B test), an LLM agent loads the page, reads the visible blocks, and extracts packages itself. One-shot by default; multi-turn loop when a dropdown or calendar needs clicking. 網站 DOM 改版、遇到新商品格式或 A/B test 時，selector 抓不到，LLM agent 自動接手——讀頁面可見內容後直接抽出 packages。預設單次抽取，需要點 dropdown 或日曆時切多輪 agent loop。
📊
Report 輸出
Everything lands in a canonical Activity → Package → SKU schema. Export to BD-ready CSV that matches the planning sheet, plus a tabbed HTML report with POI × Platform cross-tabs, hero images, real page screenshots, and filterable rows. 全部匯入 Activity → Package → SKU 三層標準結構。匯出 BD 看得懂的 CSV（欄位對齊既有表格），以及分頁的 HTML 報告——有 POI × 平台交叉表、hero 圖、實際頁面截圖、可 filter 的清單。

Features 功能特色
Everything you need for competitor analysis 競品分析所需的一切
🧱
Canonical 3-tier schema 標準化三層結構
Activity (1 per product URL) → Package (variant: language, group size, meal, vehicle) → SKU (package × travel date). Every observation gets stored so price history never overwrites. Activity（每 URL 一筆）→ Package（語言、團型、餐、車型等變體）→ SKU（package × 日期）。每次觀測都另外存一筆，歷史價格不會被覆蓋。
📝
Config-driven POIs 設定檔驅動 POI
Edit data/routine-config.json to add or remove POIs, pick which OTAs to hit, and set the per-platform limit. No code changes needed — push to git, next run uses the new list. 編輯 data/routine-config.json 加減 POI、選要跑哪幾個 OTA、設每個平台的 limit。不用改 code，push 之後下次 routine 就會用新清單。
📸
Hero + screenshot audit trail Hero 圖 + 實際截圖
Every activity stores the platform's hero image plus a real page screenshot — so you can verify the scraper actually saw the current listing, not a cached fragment. 每個 activity 都存平台 hero 圖 + 實際頁面截圖——確認 scraper 看到的真的是當下的頁面，不是 cache 的破碎片段。
🗂️
POI × Platform cross-tab POI × 平台交叉表
The Summary tab has a matrix with POIs on rows and OTAs on columns. Each cell shows "activities / SKUs" — gaps like "0 Trip.com coverage for DMZ" jump out immediately. Summary 分頁上有一張表——列是 POI、欄是 OTA，每格「活動數 / SKU 數」。一眼看出哪裡有缺口（例如「Trip.com 在 DMZ 掛零」）。
🧾
Session + execution logs Session 與執行 log
Every routine run gets a session ID; every ingest attempt logs strategy (opencli-detail / agent-browser-fallback), duration, pkgs/SKUs written, and error. Trivial to debug yesterday's failure. 每次 routine 有獨立 session ID；每次 ingest 都記 strategy（opencli-detail / agent-browser-fallback）、耗時、pkg/SKU 數量、錯誤。昨天掛哪一筆，一目了然。
🕑
Daily routine → Vercel 每日排程 → Vercel 自動更新
System cron (or a Claude Code schedule) runs the pipeline every morning, commits the regenerated CSV + HTML, and pushes to GitHub. Vercel picks up the commit and redeploys the dashboard automatically — BD's bookmark stays valid. 系統 cron（或 Claude Code schedule）每天早上跑 pipeline、commit 新的 CSV + HTML、push 到 GitHub。Vercel 自動 redeploy，BD 的 bookmark 不用改。

Getting Started 開始使用
5 Minutes to Get Started 5 分鐘快速上手
1
Install everything with one command 一行指令安裝全部

This installs opencli, klook-cli, builds the Browser Bridge extension, and registers all plugins. It will also ask for your OpenRouter API key. 自動安裝 opencli、klook-cli、建構 Browser Bridge 擴充程式、註冊所有平台。最後會詢問你的 OpenRouter API key。 

curl -fsSL https://raw.githubusercontent.com/ryanhuang1109/klook-cli/main/install.sh | bash

After install, load the Browser Bridge extension in Chrome: chrome://extensions/ → Developer Mode → Load unpacked → select /tmp/opencli-extension/extension 安裝完成後，在 Chrome 載入擴充程式：chrome://extensions/ → 開發者模式 → 載入未封裝項目 → 選擇 /tmp/opencli-extension/extension

2
Open the project in Claude Code 用 Claude Code 開啟專案目錄

Claude Code will automatically read CLAUDE.md and understand all available commands. Claude Code 會自動讀取 CLAUDE.md，了解所有可用指令。 

cd ~/klook-cli && claude
3
Set your OpenRouter API key (for AI comparison) 設定 OpenRouter API key（AI 比價功能需要）

Get your key at openrouter.ai/keys, then run: 先到 openrouter.ai/keys 取得 key，然後執行： 

mkdir -p ~/.klook-cli
echo '{"openrouter_api_key":"YOUR-KEY-HERE"}' > ~/.klook-cli/config.json

Skip this step if you already set the key during installation. 如果安裝時已經設定過，可以跳過這步。

4
Verify Browser Bridge connection 確認 Browser Bridge 已連線

Make sure Chrome is open with the Browser Bridge extension loaded. 確保 Chrome 已開啟且 Browser Bridge 擴充程式已載入。 

opencli doctor

You should see: [OK] Daemon: running, [OK] Extension: connected, [OK] Connectivity: connected 應該顯示：[OK] Daemon: running, [OK] Extension: connected, [OK] Connectivity: connected

▶ Engineer Setup Guide (8 steps) 工程師安裝指南（8 步驟）
1 Install opencli 安裝 opencli ▾

Install the opencli package globally via npm:

透過 npm 全域安裝 opencli：

npm install -g @jackwener/opencli
2 Install Browser Bridge (Chrome Extension) 安裝 Browser Bridge（Chrome 擴充程式） ▾

Clone and build the extension:

克隆並建置擴充程式：

git clone --depth 1 https://github.com/jackwener/opencli.git /tmp/opencli
cd /tmp/opencli/extension
npm install
npm run build

Then in Chrome: go to chrome://extensions/ → enable Developer Mode → Load unpacked → select /tmp/opencli/extension

然後在 Chrome 中：前往 chrome://extensions/ → 開啟開發人員模式 → 載入未封裝項目 → 選擇 /tmp/opencli/extension

3 Verify the installation 驗證安裝 ▾

Run the diagnostic command to confirm everything is connected:

執行診斷指令確認一切正常連線：

opencli doctor
4 Clone and build this repo 克隆並建置本 repo ▾ 
git clone https://github.com/ryanhuang1109/klook-cli.git
cd klook-cli
npm install
npm run build
5 Register plugins with opencli 向 opencli 登錄插件 ▾

Create symlinks so opencli can find the platform plugins:

建立符號連結讓 opencli 找到各平台插件：

ln -sf $PWD/dist/clis/klook ~/.opencli/plugins/klook
ln -sf $PWD/dist/clis/trip ~/.opencli/plugins/trip
ln -sf $PWD/dist/clis/getyourguide ~/.opencli/plugins/getyourguide
ln -sf $PWD/dist/clis/kkday ~/.opencli/plugins/kkday
6 Set OpenRouter API key 設定 OpenRouter API 金鑰 ▾

Create the config directory and write your API key:

建立設定目錄並寫入 API 金鑰：

mkdir -p ~/.klook-cli
echo '{"openrouter_api_key":"YOUR-KEY"}' > ~/.klook-cli/config.json
7 Log into platforms in Chrome 在 Chrome 登入各平台 ▾

You only need to do this once. Open Chrome and log into these sites manually:

只需要做一次。在 Chrome 手動登入以下網站：

trip.com
getyourguide.com
kkday.com

Klook uses a public API and does not require login.

Klook 使用公開 API，不需要登入。

8 Start using the tool 開始使用 ▾

Run a search to confirm everything works, then start the web dashboard:

執行搜尋確認一切正常，然後啟動 web 儀表板：

opencli klook search "Tokyo" --limit 5

# or start the web dashboard
npm run web
🔒

The Browser Bridge does not store any passwords. It works by connecting to your existing Chrome session — the same way you'd browse manually. BD colleagues using the web dashboard don't need to log into anything — the server machine's Chrome handles all access. Browser Bridge 不儲存任何密碼。它透過連接你現有的 Chrome session 來運作 — 跟你手動瀏覽完全一樣。BD 同事使用 web 介面時不需要登入任何網站，所有存取都由跑 server 的那台機器處理。
