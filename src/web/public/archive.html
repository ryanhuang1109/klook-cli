Features that used to live in this dashboard but are no longer surfaced here. Listed for reference; the underlying CLI commands still work locally.

POI Compare

CLI only

Cross-platform price comparison with LLM clustering. Used to be at /dashboard.html. The web form pointed at a local Express server, which doesn't exist on the Vercel deploy, so the page is gone. The functionality lives on in the CLI: 

node dist/cli.js list-pois
node dist/cli.js add-poi "Mt Fuji" --keywords "Mt Fuji tour" --platforms klook,trip,getyourguide,kkday
node dist/cli.js compare-poi "Mt Fuji" --date 2026-05-01 --save
node dist/cli.js get-poi-price-history "Mt Fuji" --days 7

POI config lives at ~/.klook-cli/pois.json; compare history at ~/.klook-cli/history.db. Both are local files — they aren't mirrored to Supabase yet, so coworkers can't see this data through the dashboard.

Landing & About pages

removed

The marketing-style /index.html and /about.html pages were dropped — this is an internal tool, not a public site. The dashboard root now redirects straight to the Coverage view.

Tours "Run now" form

local only

The button on /tours.html that kicks off a fresh competitor scan calls a local Express endpoint that spawns a Node child process — neither exists on Vercel, so the form is hidden in production. Run from the CLI instead: 

node dist/cli.js tours run --destination tokyo --keyword "mt fuji" \
  --competitors klook,trip,getyourguide,kkday --limit 10 --poi "Mt Fuji"

Once the local run finishes, tours sync-to-supabase publishes the results so they appear on the live Tours / Runs / Executions views.
